Modules are self-contained scripts that handle specific setup tasks.

## Module Template

```bash
#!/bin/bash

# natilius - Module Name
# Brief description of what this module does
#
# Copyright (C) 2024 Vincent Koc (@vincent_koc)
# License: GPL-3.0-or-later

# ============================================================================
# MODULE: category/module_name
# Description: What this module does
# Dependencies: homebrew (list required modules)
# Config: VARIABLE_NAME (list config variables used)
# ============================================================================

log_info "Starting module_name setup..."

# ============================================================================
# Prerequisites
# ============================================================================

# Check for required commands
if ! command -v required_tool &> /dev/null; then
    log_warning "required_tool not found, skipping module"
    return 0
fi

# ============================================================================
# Main Logic
# ============================================================================

install_something() {
    if some_check; then
        log_success "Already installed, skipping"
        return 0
    fi

    log_info "Installing something..."
    # Installation logic
    log_success "Something installed"
}

configure_something() {
    log_info "Configuring something..."
    # Configuration logic
    log_success "Something configured"
}

# ============================================================================
# Execute
# ============================================================================

install_something
configure_something

log_success "module_name setup complete"
```

## Guidelines

### 1. Be Idempotent

Modules must be safe to run multiple times:

```bash
# Good - check before installing
if ! command -v tool &> /dev/null; then
    install_tool
fi

# Good - check version
if [[ "$(tool --version)" != "$EXPECTED" ]]; then
    upgrade_tool
fi
```

### 2. Check Prerequisites

Don't assume dependencies exist:

```bash
if ! command -v brew &> /dev/null; then
    log_error "Homebrew required but not installed"
    return 1
fi
```

### 3. Use Logging Functions

```bash
log_info "Starting..."     # Blue, informational
log_success "Done!"        # Green, success
log_warning "Skipping..."  # Yellow, warning
log_error "Failed!"        # Red, error
```

### 4. Handle Errors Gracefully

Don't crash the entire setup:

```bash
# Use || true for non-critical failures
some_command || true

# Or handle explicitly
if ! some_command; then
    log_warning "some_command failed, continuing"
fi
```

### 5. Document Dependencies

In the module header:

```bash
# Dependencies: applications/homebrew, dev_environments/python
# Config: MY_VERSION, MY_PACKAGES
```

### 6. Use Config Variables

Read from user config:

```bash
# Use defaults if not set
VERSION="${MY_VERSION:-1.0.0}"

# Iterate arrays
for pkg in "${MY_PACKAGES[@]}"; do
    install_package "$pkg"
done
```

## Module Locations

| Category | Path | Purpose |
|----------|------|---------|
| `system/` | `modules/system/` | OS-level operations |
| `applications/` | `modules/applications/` | Package/app management |
| `dev_environments/` | `modules/dev_environments/` | Language setups |
| `ide/` | `modules/ide/` | Editor configurations |
| `preferences/` | `modules/preferences/` | System preferences |

## Adding Your Module

<Steps>
  <Step title="Create the module">
    ```bash
    touch modules/category/my_module.sh
    chmod +x modules/category/my_module.sh
    ```
  </Step>
  <Step title="Write the code">
    Use the template above.
  </Step>
  <Step title="Add to module list">
    Update `natilius.sh` to include in `list-modules`.
  </Step>
  <Step title="Document">
    Add to `docs/api-reference/modules.mdx`.
  </Step>
  <Step title="Test">
    ```bash
    # Add to your config
    ENABLED_MODULES+=("category/my_module")

    # Test
    natilius --check
    natilius setup
    ```
  </Step>
  <Step title="Write tests">
    Add tests in `tests/`.
  </Step>
</Steps>

## Example: Custom Tool Module

```bash
#!/bin/bash

# natilius - MyTool Setup
# Installs and configures MyTool

# ============================================================================
# MODULE: applications/mytool
# Description: Installs MyTool CLI
# Dependencies: applications/homebrew
# Config: MYTOOL_VERSION
# ============================================================================

log_info "Setting up MyTool..."

MYTOOL_VERSION="${MYTOOL_VERSION:-latest}"

# Check if already installed
if command -v mytool &> /dev/null; then
    current=$(mytool --version)
    log_success "MyTool already installed ($current)"

    if [[ "$MYTOOL_VERSION" == "latest" ]]; then
        return 0
    fi
fi

# Install
log_info "Installing MyTool $MYTOOL_VERSION..."
brew install mytool || {
    log_error "Failed to install MyTool"
    return 1
}

# Configure
log_info "Configuring MyTool..."
mytool configure --default

log_success "MyTool setup complete"
```
