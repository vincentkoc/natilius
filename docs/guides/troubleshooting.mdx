## Quick Diagnostics

Always start with:

```bash
natilius doctor
```

This checks macOS version, Xcode CLI tools, Homebrew, config validity, and more.

## Installation Issues

<AccordionGroup>
  <Accordion title="Command not found: natilius">
    The command isn't in your PATH.

    ```bash
    # Restart shell
    exec $SHELL

    # Or add to PATH
    export PATH="/usr/local/bin:$PATH"

    # Verify
    which natilius
    ```
  </Accordion>

  <Accordion title="Permission denied">
    Can't write to `/usr/local/bin`.

    ```bash
    sudo chown -R $(whoami) /usr/local/bin
    ```
  </Accordion>

  <Accordion title="Homebrew not found (Apple Silicon)">
    On M1/M2/M3, Homebrew is at `/opt/homebrew`.

    Add to `~/.zshrc`:
    ```bash
    eval "$(/opt/homebrew/bin/brew shellenv)"
    ```

    Then: `source ~/.zshrc`
  </Accordion>
</AccordionGroup>

## Runtime Issues

<AccordionGroup>
  <Accordion title="sudo: a password is required">
    Running non-interactively but sudo needs a password.

    **Options:**
    1. Run interactively: `natilius setup`
    2. Use `SKIP_SUDO=true` (skips security module)
    3. Configure passwordless sudo (not recommended)
  </Accordion>

  <Accordion title="Module fails: command not found">
    A dependency isn't installed.

    Ensure `homebrew` runs before dev environments:
    ```bash
    ENABLED_MODULES=(
        "applications/homebrew"  # First!
        "dev_environments/python"
    )
    ```
  </Accordion>

  <Accordion title="Python/Node version not available">
    The requested version doesn't exist.

    ```bash
    # Update version managers
    brew upgrade pyenv nodenv

    # List available versions
    pyenv install --list | grep "3.12"
    nodenv install --list

    # Use an available version
    PYTHONVER="3.12.1"
    ```
  </Accordion>
</AccordionGroup>

## Apple Silicon Issues

<AccordionGroup>
  <Accordion title="Rosetta 2 errors">
    Some tools need Intel compatibility.

    ```bash
    softwareupdate --install-rosetta --agree-to-license
    ```
  </Accordion>

  <Accordion title="Bad CPU type in executable">
    Running an Intel binary without Rosetta.

    ```bash
    # Run with Rosetta
    arch -x86_64 /path/to/binary

    # Or reinstall ARM version
    brew reinstall <package>
    ```
  </Accordion>
</AccordionGroup>

## Config Issues

<AccordionGroup>
  <Accordion title="Invalid version format">
    Version strings must be full semantic versions.

    ```bash
    # Wrong
    PYTHONVER="3.11"

    # Correct
    PYTHONVER="3.11.0"
    ```
  </Accordion>

  <Accordion title="Module not found">
    Check spelling against `natilius list-modules`.

    ```bash
    # Wrong
    "dev-environments/python"

    # Correct
    "dev_environments/python"
    ```
  </Accordion>
</AccordionGroup>

## Network Issues

<AccordionGroup>
  <Accordion title="Could not resolve host">
    DNS or connectivity issues.

    ```bash
    # Test connectivity
    ping -c 1 github.com

    # Try different DNS
    sudo networksetup -setdnsservers Wi-Fi 8.8.8.8 8.8.4.4
    ```
  </Accordion>

  <Accordion title="Corporate proxy">
    ```bash
    export http_proxy="http://proxy:port"
    export https_proxy="http://proxy:port"
    git config --global http.proxy http://proxy:port
    ```
  </Accordion>
</AccordionGroup>

## Debug Mode

Get detailed output:

```bash
natilius -v setup 2>&1 | tee debug.log
```

## Check Logs

```bash
ls -la ~/.natilius/logs/
cat ~/.natilius/logs/natilius-setup-*.log | tail -100
```

## Reset and Start Fresh

```bash
# Uninstall
~/.natilius/uninstall.sh

# Remove config
rm ~/.natiliusrc

# Reinstall
curl -fsSL https://raw.githubusercontent.com/vincentkoc/natilius/main/install.sh | bash
```

## Get Help

2. Check [GitHub Issues](https://github.com/vincentkoc/natilius/issues)
