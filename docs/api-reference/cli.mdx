## Synopsis

```bash
natilius [command] [options]
```

## Commands

### setup

Run the full setup process (default command).

```bash
natilius setup
natilius  # same as above
```

### doctor

Run system diagnostics and check readiness.

```bash
natilius doctor
```

Checks:
- macOS version and architecture
- Xcode Command Line Tools
- Homebrew installation
- Git availability
- Configuration validity
- Disk space
- Network connectivity
- Sudo access
- Rosetta 2 (Apple Silicon)

### list-modules

Show all available modules.

```bash
natilius list-modules
```

### version

Show version information.

```bash
natilius version
```

### help

Show help message.

```bash
natilius help
natilius --help
natilius -h
```

## Options

### -p, --profile NAME

Use a specific configuration profile.

```bash
natilius --profile devops
natilius -p minimal
```

Loads `~/.natiliusrc.NAME` instead of `~/.natiliusrc`.

### -c, --check, --dry-run

Preview changes without making them.

```bash
natilius --check
natilius --dry-run
natilius -c
```

### -i, --interactive

Run in interactive mode with module selection prompts.

```bash
natilius --interactive
natilius -i
```

### -v, --verbose

Enable detailed output.

```bash
natilius --verbose
natilius -v
```

### -q, --quiet

Suppress non-error output.

```bash
natilius --quiet
natilius -q
```

## Examples

```bash
# Basic setup
natilius

# Preview changes
natilius --check

# Use DevOps profile
natilius --profile devops

# Interactive selection
natilius --interactive

# Quiet mode for automation
natilius --quiet setup

# Verbose for debugging
natilius -v setup

# Combine options
natilius -v --profile developer --check
```

## Exit Codes

| Code | Meaning |
|------|---------|
| `0` | Success |
| `1` | Error (check logs for details) |
